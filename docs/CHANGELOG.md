## 版本说明

### 版本号规则

项目采用语义化版本控制 (Semantic Versioning):

- **主版本号**: 不兼容的API修改
- **次版本号**: 向下兼容的功能性新增
- **修订版本号**: 向下兼容的问题修正

### 变更类型

- ✨ **新功能** (Features): 新增功能
- 🐛 **Bug修复** (Bug Fixes): 问题修复
- 🔧 **技术改进** (Technical): 代码重构、性能优化
- 📝 **文档更新** (Documentation): 文档相关变更
- 🔐 **安全更新** (Security): 安全相关修复
- 💥 **破坏性变更** (Breaking Changes): 不兼容的变更

### 发布周期

- **主版本**: 根据重大功能更新发布
- **次版本**: 每月发布，包含新功能和改进
- **修订版本**: 根据Bug修复需要随时发布
